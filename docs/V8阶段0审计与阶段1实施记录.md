- DeepSeek 流式代理改为只发送 delta，不再反复发送累计全文。

- Dashboard 订阅六张真实学习事实表，数据变化后 250ms 合并刷新。
- Dashboard 显示最近一次真实数据更新时间。
- 新增 Realtime publication 数据库迁移。
- 修复 SmartRecommend 对 PromiseSettledResult 的错误解构。
- 修复数字人连接按钮不可能状态比较导致的 TypeScript 错误。

- search-knowledge 与 system-health 已彻底删除 DEMO_MODE / DEMO_RESULTS 生产回退。
- 新增真实活跃学习计时器：仅页面可见且用户近期操作时累计，完整分钟才落库。

- 新增海报式 ResourcePosterWall，已嵌入课程知识点与错题本底部。
- 外部资源优先站内 iframe 预览；站点禁止嵌入时提供安全外跳。

- 管理后台资源审核页新增“联网发现候选”入口，可按查询词、知识主题和数量发起发现。
- 资源墙新增收藏、标记学完、不感兴趣三类反馈；不感兴趣后立即从当前推荐中移除。
- 考前冲刺页已接入考试场景资源墙，使用当前输入和最近对话主题参与匹配。
- 新增右下角常驻 AI 学习管家，在所有登录后页面可用，并携带当前模块上下文。

- 新增 learning_tasks、learning_plans 两张真实业务表；无数据时保持为空，不注入演示任务或计划。
- 新增 assistant_action_requests 与 assistant_action_audit，记录操作提议、确认执行、拒绝与失败。
- 首批受控工具为创建学习任务、创建/修改学习计划、标记错题掌握状态。
- 所有写操作先生成确认卡片，用户确认后服务端按 JWT 用户范围执行；重复确认返回冲突，不重复写入。
- 新增任务与计划中心 `/tasks`，显示真实任务和学习计划，支持手动建任务、完成任务与取消任务；空库时显示空状态，不生成演示数据。
- 任务与计划中心订阅 learning_tasks、learning_plans，数据库变更后 250ms 合并刷新。
- Dashboard 新增真实待办任务指标，并订阅任务、计划数据，实现 1–3 秒内联动更新。
- 学习管家已增加 10 分钟限时撤销：新建任务回退为取消、创建计划回退为取消、修改计划恢复修改前版本、错题状态恢复原状态。
- 正向执行和撤销均写入 assistant_action_audit；请求状态增加 undone，避免重复执行和重复撤销。
- 新增 00016_learning_butler_undo.sql，为操作请求增加撤销窗口与撤销时间，并将操作请求加入 Realtime publication。
- 新增 `/butler-audit` 管家操作记录页，按待确认、已执行、已拒绝、失败、已撤销筛选当前用户的真实记录。
- 操作记录页同时读取 assistant_action_requests 与 assistant_action_audit，展示操作摘要、风险等级、审计事件数、失败原因和撤销截止时间。
- assistant_action_requests 与 assistant_action_audit 均加入 Realtime，操作状态与审计事件在 1–3 秒内刷新。
- Edge Function 新增服务端工具策略白名单：创建任务、修改计划为中风险，更新错题状态为低风险；三者均强制确认并提供 10 分钟撤销。
- 个人知识库原有浏览器 `includes` 文本匹配已替换为服务端混合检索，不再把简单字符串包含伪装成 RAG。
- 新增 00017_wvk_rag.sql：包含 Wiki 词条、词条关系、知识片段、索引任务、RLS、检索索引和服务端专用混合检索 RPC。
- 新增 index-knowledge-wiki Edge Function：把用户已解析资料整理为结构化 Wiki 词条和关系，同时建立 RAG 片段；全过程校验 JWT 和文件归属。
- 新增 knowledge-retrieval-agent Edge Function：并行召回 WVK 结构词条、全文/中文模糊匹配和可选向量结果，再按知识关系重排。
- 向量模型作为检索子能力封装在服务端函数中；配置 1024 维 OpenAI 兼容 Embedding 后启用向量通道，未配置时明确返回 `wvk+fts`，不会伪造向量结果。
- 知识库页面新增 WVK 索引/重建入口、真实索引状态、Wiki/片段/向量计数、检索模式、结果来源与分数。
- 重建索引会检查删除与写入错误；词条删除通过外键级联清理关系，避免空 `IN()` 导致不确定行为。

## 3. 审计发现

### 真实流式

- Chat、Tutor、ExamCram 和对应 Edge Function 都调用上游流式接口，并已统一前端增量解析。
- 原 Chat 前端与 deepseek-chat-stream 后端按单个网络块直接拆行，存在跨块 JSON 丢失风险。
- Tutor 已有半行 buffer，但未形成全站统一协议。
- Chat 与 ExamCram 已支持中止旧请求；Tutor 后续补充显式“停止生成”交互。

### 数据真实性

- Dashboard 指标主要来自 learning_records、learning_progress、learning_events、knowledge_mastery、learning_profiles 和 agent_runs。
- Chat/Tutor 存在固定学习时长写入，现已删除。
- search-knowledge 与健康检查中的 Demo 分支已经清除；配置缺失时返回 503/空结果或 down。
- 无数据页面应统一显示 0 或明确空状态。

### 实时性

- 原 Dashboard 仅首次进入和手动刷新时加载。
- 本轮已增加 Realtime 订阅；线上还必须执行 00013 迁移。
- 课程完成、答题、错题和助手写操作仍需逐项验证是否都会产生事实事件。

## 5. 部署注意事项

- 先备份数据库，再执行 00013_enable_learning_realtime.sql。
- 再执行 00014_recommendation_loop.sql；迁移不会插入任何演示资源。
- Supabase SQL Editor 必须粘贴并运行 00014_recommendation_loop.sql 的完整内容；只运行文件名注释不会创建任何对象。
- 部署 discover-learning-resources Edge Function，并在 Supabase 服务端 Secret 中配置 TAVILY_API_KEY。
- 发现函数只写入 pending 状态，禁止绕过管理员审核直接公开。
- 执行 00015_learning_butler.sql 后再部署 learning-butler Edge Function。
- 线上已由用户截图确认 00015 的 learning_tasks、learning_plans、assistant_action_requests、assistant_action_audit 四张表均已创建。
- 下一步执行完整的 00016_learning_butler_undo.sql，然后重新部署 learning-butler Edge Function 与前端；只执行文件名不会应用迁移。
- Supabase Edge Function Secret 配置 SPARK_API_PASSWORD；兼容已有 SPARK_API_KEY + SPARK_API_SECRET 组合，但不得下发前端。
- 学习管家上线验收必须覆盖：未登录 401、跨用户资源不可修改、拒绝不写业务表、重复确认不重复执行、失败操作有审计记录。
- 确认所有表的 RLS 仍只允许用户读取自己的数据。
- Realtime 失败不得回退为假数字，只允许显示最后更新时间与连接状态。
- 所有模型、TURN、Supabase Service Role 和数字人凭据仅使用服务端 Secret。
- 执行完整的 00017_wvk_rag.sql；随后部署 index-knowledge-wiki 与 knowledge-retrieval-agent 两个 Edge Function。
- Wiki 整理依赖服务端 SPARK_API_PASSWORD。浏览器不得读取 Spark、Embedding 或 Service Role 密钥。
- 完整向量通道需配置 EMBEDDING_API_URL、EMBEDDING_API_KEY、EMBEDDING_MODEL；数据库向量维度固定为 1024，模型输出维度不符时拒绝写入并明确退回 WVK + FTS。
- 00017 验收顺序：上传并解析资料 → 点击 WVK 索引 → 等待 completed → 确认 Wiki/片段计数大于 0 → 搜索并确认来源/分数；没有 Embedding 配置时向量计数应真实显示 0。
- 00017 已兼容 Supabase PostgreSQL 的严格生成列限制：全文 `tsvector` 改由数据库触发器自动维护并回填，别名和关键词仍进入索引；迁移会恢复此前失败留下的生成列半应用状态。
