## 2. 状态定义

- **代码已写**：本地已有实现，且至少通过静态检查或构建。
- **已部署**：用户明确反馈已执行线上 SQL、Edge Function 或服务部署。
- **已验收**：用户实际操作并确认结果正确。
- 不得把“存在文件”直接视为“已经部署并验收”。

## 4. 已完成或已有代码的能力

### 4.1 真实流式输出

- Chat、Tutor、ExamCram 已实现 SSE 增量解析。
- 前端按 `delta` 追加内容，收到 `done` 后结束。
- 流式接口带用户身份令牌，不再仅做客户端打字机动画。
- 仍需在生产环境通过慢速网络实际观察首字延迟、持续增量和中断恢复。

### 4.2 实时学习数据

- Dashboard 订阅真实事实表并执行约 250ms 合并刷新。
- 课程测验、章节完成、学习时长等已有真实事件写入入口。
- 已取消 Dashboard 演示数据回退；无数据时应显示 0。
- 实时目标为关键状态 1–3 秒内更新。

### 4.3 智能资源推荐

- 推荐资源数据库、管理员审核、推荐反馈和场景关联已建立。
- `discover-learning-resources` Edge Function 已存在。
- 用户明确反馈：函数已部署，`TAVILY_API_KEY` 已配置。
- 课程章节、考试错题和错题本页面已有海报式推荐墙入口。
- 外链优先站内预览，无法嵌入时再跳转原站。

### 4.4 常驻 AI 学习管家

- 已设计并实现 `learning_tasks`、`learning_plans`、`assistant_action_requests`、`assistant_action_audit`。
- 用户截图确认上述四张表已存在。
- AI 可提出创建任务、修改计划、修改错题状态等受控操作。
- 高风险写操作需要确认，并保留审计记录；部分操作支持限时撤销。
- 常驻助手使用 Spark Lite；数字人仅在 GPU 在线或演示模式下优先启用。

### 4.5 数字人原型

- AutoDL 4090 环境中的 MuseTalk/LiveTalking 已运行。
- WebRTC 已切换到带认证的公网 Coturn，UDP/TCP relay 测试通过。
- 用户已实测：有画面、文本可驱动语音、口型同步有效。
- 该结果属于独立数字人服务原型，尚不等于已嵌入智学帮生产前端。

### 4.6 WVK + RAG 第一批

- 数据库迁移 `00017` 已准备，用户最后反馈“跑通了”。
- 已有 Wiki 条目、关系、知识块、索引任务和混合检索 RPC。
- `index-knowledge-wiki` 负责知识结构化与索引。
- `knowledge-retrieval-agent` 负责双路召回、可选向量召回和重排。
- 知识库页面已有检索模式、引擎计数、得分和来源展示区域。

## 5. 上一轮准确停点

停在以下前后端字段不完全对齐的问题：

- 前端 `src/pages/KnowledgePage.tsx` 已读取：
  - `sourceName`
  - `sourceUrl`
  - `relationCount`
  - `engine_summary`
- 后端 `supabase/functions/knowledge-retrieval-agent/index.ts` 当前主要返回：
  - `query`
  - `mode`
  - `answer`
  - `results`
  - `relations`
  - `vector_enabled`
- 后端尚未稳定为每条结果补齐规范化的 `source_name`、`source_url`、`relation_count`，也未返回统一的 `engine_summary`。

因此界面虽已准备显示“证据和来源”，但真实查询时可能只能显示回退值。下一位 AI 应从这里继续，避免重新设计整个 WVK/RAG 模块。

## 8. 数据库复核 SQL

不要仅凭迁移文件判断部署成功，可在 Supabase SQL Editor 执行只读检查：

```sql
select
  to_regclass('public.wiki_entries') as wiki_entries,
  to_regclass('public.wiki_relations') as wiki_relations,
  to_regclass('public.knowledge_chunks') as knowledge_chunks,
  to_regclass('public.knowledge_index_jobs') as knowledge_index_jobs,
  to_regprocedure(
    'public.hybrid_knowledge_search(uuid,text,extensions.vector,integer)'
  ) as hybrid_search;
```

若对象已存在，不要再次整段执行旧迁移。新增修复应创建新的增量迁移，避免重复列、重复策略和生成列表达式错误。
